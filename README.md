# Cultural Motifs Repo

A structured repository for documenting cultural visual motifs in a way that is useful for:

- visual research
- AI prompting
- shader systems
- generative art systems
- app integration
- drift prevention

This is **not** a generic "world patterns" moodboard repo.

The purpose of this repository is to organize motif systems with enough specificity that they remain useful, distinguishable, and machine-tractable. The repo is designed to help artists, researchers, and software systems work with cultural motifs without collapsing them into decorative soup.

---

## Core goals

This repo is meant to support:

1. **Style definition**
   - identifying recurring visual motif families
   - describing formal traits, composition logic, and symbolic tendencies

2. **Prompting**
   - generating more accurate and less generic prompts for image, video, and design systems

3. **Generative systems**
   - translating motifs into repeat logic, symmetry logic, border systems, icon placement, and compositional rules

4. **App integration**
   - exposing motif families, cultural tags, material traits, and comparison structures in machine-readable formats

5. **Drift prevention**
   - preventing the flattening of distinct traditions into vague "ancient / ethnic / ornate" filler

---

## What this repo is

This repo is:

- a motif atlas
- a visual analysis framework
- a prompt and generative-art reference system
- a structure for comparing similar-looking motifs across traditions without pretending they mean the same thing
- a tool for keeping cultural specificity intact

---

## What this repo is not

This repo is **not**:

- a random ornament dump
- a tourist-shop version of world culture
- a fake archaeology generator
- a fantasy lore wiki
- a substitute for deep historical scholarship
- a justification for careless mashups

This repo should not drift into phrases like:

- "tribal patterns"
- "exotic motifs"
- "oriental ornament"
- "ancient energy symbols"
- "mystic ethnic geometry"

If the language starts sounding like that, something has gone wrong.

## Design principle

The repo separates the following whenever possible:

- **visual form**  
  what the motif looks like

- **symbolic role**  
  what the motif may represent or do

- **material context**  
  where the motif appears physically and what media it belongs to

- **compositional role**  
  how it behaves in borders, panels, fields, architecture, garments, surfaces, or scenes

- **confusion risks**  
  what it is likely to be mistaken for

This separation matters because motifs that look similar across traditions often do **not** carry identical meanings or functions.

## Repo logic

The repository is organized into several major layers:

- **scope and method**
- **foundation**
- **taxonomy**
- **regions and cultures**
- **cross-cultural motif families**
- **formal visual logic**
- **materials, color, and surface**
- **symbolic roles and meaning**
- **folklore and specific beings**
- **drift control**
- **prompt system**
- **shader and generative-art translation**
- **app integration**
- **structured data**
- **examples**

The repo uses both:
- **culture-specific folders** for precision
- **cross-cultural comparison files** for controlled comparison

This avoids two bad outcomes:
1. one giant filing cabinet with no comparison logic
2. one giant comparison soup with no specificity

## Content standards

Good files in this repo should be:

- clear
- structured
- specific
- conservative when uncertain
- explicit about ambiguity
- machine-usable
- anti-drift aware

Bad files in this repo tend to be:

- vague
- overconfident
- decorative but empty
- fantasy-coded
- overly mystical without evidence
- lazy about cultural distinctions

---

## Guidance for contributors and agents

When adding or expanding files:

- prefer specificity over vibes
- distinguish visual traits from symbolic meaning
- do not invent symbolism
- do not flatten adjacent cultures
- include confusion risks
- use stable headings and structures
- keep markdown clean
- keep JSON valid
- keep CSV valid

Read these files before generating content:

- `REPO_STRUCTURE.md`
- `AGENT_BUILD_PLAN.md`
- `AGENT_GENERATION_RULES.md`
- `FILE_TEMPLATES.md`

If exemplar files exist, match their structure closely.

---

## Recommended workflow

1. define the scope of a file clearly
2. describe formal traits
3. note common motif families
4. describe compositional and material behavior
5. note symbolic tendencies cautiously
6. identify confusion risks
7. add prompting / genart implications where useful
8. add or update structured data only after the analysis layer is coherent

---

## Warning about hybridization

This repo does not forbid comparison or hybridization, but it treats both as **controlled operations**, not defaults.

A careful hybrid:
- names its sources explicitly
- preserves their distinctions
- avoids turning motifs into vague decorative garnish

A bad hybrid:
- blends "Egyptian + Mayan + Islamic + Japanese" into an ahistorical costume slurry
- treats sacred or culturally specific motifs like random sticker packs
- relies on broad vibe words instead of formal structure

See:
- `09_drift_control/`
- `10_prompt_system/careful_hybridization.md`
